# Сервис ставок

## Назначение

Bidding Service отвечает за приём bid requests от DSP-партнёра, подбор рекламных кампаний, соответствующих таргетингу, и расчёт оптимальной ставки для участия в RTB-аукционе.

Сервис выделяется из текущего монолита, так как он находится на критическом пути (hot path) и предъявляет самые жёсткие требования к латенции.

## Границы сервиса

### Что входит в ответственность Bidding Service

- Приём HTTP-запросов по протоколу OpenRTB через API Gateway
- Валидация входящего bid request
- Чтение активных кампаний из Redis-кэша
- Фильтрация кампаний по таргетингу (гео, устройство, время суток, бюджет)
- Вычисление ставки (на основе правил кампании, остатка бюджета, исторического CTR)
- Передача кандидатов в Auction Engine (через gRPC) для финального взвешивания
- Возврат bid response

### Что НЕ входит в ответственность Bidding Service

- Финальное определение победителя аукциона (это делает Auction Engine)
- Запись событий (клики, показы) — это асинхронно через Stats Writer
- Управление кампаниями (CRUD) — это ответственность Campaign Service
- Финансовые списания — ответственность Payment Service

### Зависимости от других сервисов

| Зависимость | Тип | Назначение | SLA требования |
|-------------|-----|------------|----------------|
| Redis (кэш кампаний) | Хранилище | Чтение активных кампаний, ставок, таргетинга | < 5 мс, p99 |
| Auction Engine | Синхронный gRPC | Взвешивание кандидатов, определение победителя | < 20 мс, p99 |
| Campaign Service | Асинхронный (через Kafka) | Получение обновлений кампаний для инвалидации кэша | Eventual consistency |

**Важно:** Bidding Service не вызывает Campaign Service синхронно в hot path. Обновления кампаний поступают через Kafka-события, которые обновляют Redis-кэш асинхронно.

## API сервиса

### Внешний API (через API Gateway)

Bidding Service не экспортирует HTTP-API напрямую во внешнюю сеть. Все внешние запросы проходят через API Gateway.

События, которые публикует Bidding Service (в Kafka)
Событие	Топик	Назначение
BidRequestReceived	bidding.events	
Аудит, мониторинг, аналитика
BidMade	bidding.events	
Фиксация факта участия в аукционе
Модель данных
Bidding Service не имеет собственной базы данных. 
Вся оперативная информация хранится в Redis-кэше.
