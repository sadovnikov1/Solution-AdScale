2. Целевое архитектурное видение (через 12 месяцев)
Бизнес-цели к целевому состоянию:

50 000 RPS

SLA 99,9% (доступность), P95 latency ≤ 100 мс

Мультирегиональное развёртывание

Микросервисная, событийно-ориентированная архитектура

Ключевые архитектурные принципы целевого решения:

Разделение данных – OLTP, аналитика, финансы – на отдельные хранилища.

Горизонтальное масштабирование каждого сервиса независимо – через контейнеризацию (Kubernetes) и stateless сервисы.

Асинхронная обработка событий (клики/показы) через очереди сообщений (Kafka / RabbitMQ).

Кэширование в hot path – Redis для ставок, кампаний, user profile.

Изоляция критического пути аукциона – Auction Engine работает только с in-memory данными и кэшем, без прямых вызовов БД.

CQRS + Event Sourcing для финансов и статистики (опционально).

Микросервисы вместо монолита – с выделением: Bidding Service, Stats Service, Analytics Service, Payment Service, Campaign Management Service.

Автоматизированный деплой и SRE-практики – мониторинг (Prometheus + Grafana), алертинг, circuit breakers.
